⚔ Pokémon Battle Comparison Tool

Documentation and Usage Guide

Project Overview

This interactive D3.js v3 visualization project allows users to compare any two Pokémon from all generations based on their stats and types. It provides a clear battle comparison with dynamic charts and filtering options.

Technologies Used

  • D3.js v3 for data visualization
  • HTML5
  • CSS3
  • JavaScript ES5/ES6
  • JavaScript Object Dataset (pokemon_data.js)

Features

  • Filter Pokémon by Type, Generation, and Legendary status
  • Select two Pokémon to compare their stats side-by-side
  • Visualize stats comparison using D3.js v3 bar charts
  • Dynamic battle result prediction based on weighted stats
  • User-friendly interface with Pikachu-yellow themed styling

File Structure

pokemon-battle-tool/
├── index.html          # Main application interface and logic
├── pokemon_data.js     # Pokémon dataset with all generations and stats
└── documentation.html  # This documentation file

Setup Instructions

  1. Ensure all files are in the same folder.
  2. Open index.html in a modern web browser (Chrome, Firefox, Safari, Edge).
  3. The app will load the Pokémon data and initialize the interface.
  4. Use the filter controls to narrow down Pokémon choices.
  5. Select two Pokémon from the dropdown menus to compare.
  6. View stats, types, battle result, and the D3.js bar chart.

Data Structure

The pokemon_data.js file contains an array of Pokémon objects with the following structure:

[
  {
    id: Number,
    name: String,
    types: [String, ...],
    generation: Number,
    legendary: Boolean,
    stats: {
      hp: Number,
      attack: Number,
      defense: Number,
      special_attack: Number,
      special_defense: Number,
      speed: Number
    }
  },
  ...
]

Troubleshooting

  • Pokémon data not loading: Ensure pokemon_data.js is in the same folder and correctly linked in index.html.
  • Charts not displaying: Use a modern browser with JavaScript enabled and internet connection for D3.js CDN.
  • Filters not working: Check browser console for JavaScript errors.
